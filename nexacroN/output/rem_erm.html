rem / em 단위 비교 샘플

HTML CSS (rem · em) vs NexacroN XCSS (-nexa- prefix) 비교용 페이지  |  브라우저 기본 font-size = 16px (1rem = 16px)

NexacroN XCSS 참조 (비교 기준)
/* NexacroN XCSS 형태 — 공통 컴포넌트 스타일 */ -nexa-padding : 0.5em 1rem; /* em = 해당 컴포넌트 폰트 기준, rem = 루트 기준 */ -nexa-border : 0.1em solid #c2c2c2; /* 테두리 두께도 폰트 연동 */ border-radius : 0.25rem; /* 루트 기준 고정 곡률 */ font : normal 1rem/normal "Arial"; /* 절대 크기, 루트 기준 */ letter-spacing : 0.05em; /* 폰트 크기에 비례 */ word-spacing : 0.1em; /* 폰트 크기에 비례 */
em vs rem 핵심 차이 — 인터랙티브 데모

🔵 font-size: 1rem → 항상 16px (루트 기준, 부모 무관)

🔴 font-size: 1em → 부모 font-size에 연동 (슬라이더 변경 시 같이 변함)

🟢 font-size: 1rem + letter-spacing: 0.05em → 폰트 고정, 자간은 폰트 연동

※ 슬라이더를 움직이면 부모 font-size가 변합니다. rem은 변하지 않고 em은 부모를 따라갑니다.

버튼 1 — XCSS 동일 패턴

font: normal 1rem/normal "Arial"; padding: 0.5em 1rem; /* ← XCSS와 동일 */ border: 0.1em solid #c2c2c2; border-radius: 0.25rem; letter-spacing: 0.05em; word-spacing: 0.1rem;

버튼 2 — 폰트만 크게 (1.25rem)

버튼 3 — 폰트만 작게 (0.75rem)

= 16px (기본) 		현재 요소(또는 부모) font-size 속성별로 선택적 적용
font-size ✅ 전역 일관성 ⚠️ 중첩 시 누적 축소/확대 rem 권장 (전역 고정)
padding 루트 기준 고정 폰트 크기에 비례 상하: em / 좌우: rem
※ XCSS: -nexa-padding: 0.5em 1rem
border 고정 두께 (0.0625rem=1px) 폰트에 비례 (0.1em) em 권장 (폰트와 균형 유지)
※ XCSS: -nexa-border: 0.1em solid
letter-spacing 고정값 폰트에 비례 ✅ 권장 em 권장
※ XCSS: 0.05em
border-radius ✅ 고정 곡률 권장 폰트에 비례 rem 권장 (디자인 일관성)
※ XCSS: 0.25rem
